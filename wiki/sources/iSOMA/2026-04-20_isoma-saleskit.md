## Summary

iSOMA is FPT IS's comprehensive document digitization solution, developed by the IDA (Innovation & Digital Assets) division. It converts physical documents (paper, image, audio) into structured digital data through an 8-step workflow combining AI-OCR, human operators, and specialized scanning equipment. The solution targets both government ministries (construction, natural resources, home affairs, archives) and enterprise sectors (banking, insurance, retail, manufacturing, healthcare).

The saleskit (Aug 2024, 37 slides, VI) positions iSOMA as an end-to-end digitization service rather than just software — FPT provides the people, devices, technology, and facilities. The brochure (EN, 16 pages) provides the external-facing summary. The saleskit also showcases a separate Digital Processing Service for content creation (video, AR, 3D) using digitized source data.

iSOMA feeds into [[ArchiveNex]] for long-term electronic storage and retrieval. The two products together form FPT IS's full document lifecycle: digitize (iSOMA) → archive & retrieve (ArchiveNex).

## Key facts

- **Scale:** 100+ global customers; 300+ million records/documents processed; 2,000+ personnel nationwide
- **Compliance:** ISO 4001:2018; Circular 02/2019/TT-BNV; Circular 01/2023/TT-VPCP; Circular 13/2023/TT-BNV; PM Decision 458/QĐ-TTg (State electronic archive plan 2020–2025)
- **Document types supported:** A0, A2, A3, A4 and specialized formats; paper, image, PDF, audio
- **8-step process:** Collect → Classify → Scan → Check document → OCR & Data entry → Check data entry → Export & store → Search & retrieve
- **Scanning equipment:** Auto scanner, flatbed scanner, book scanner A4–A0, specialized scanner
- **Quality control:** 2-round check — Round 1: 100% output files; Round 2: random 30%
- **Output format:** PDF (default) or customer-specified format
- **Architecture:** 3-tier (iSOMA web presentation layer / web-services application layer / MariaDB + Elasticsearch + file storage data layer); integrations: Digital Signature, AI OCR, external DB/systems
- **Smart digitization model:** AI-OCR scans and auto-classifies into attribute-based folders; sensitive data protected via access control + encryption; version management; exports to CSV/TSV
- **Setup time:** 3–5 working days to set up digitization process
- **Map data (Japan):** 800+ staff processing map data for a major Japanese client since 2015; output = Google Maps base layer + autonomous vehicle HD maps; drone/aerial photography from 2018 for 3D accuracy

## Use cases documented (11)

1. **User manual digitization** — OCR + short guide videos; customer service calls reduced >50%; product sales +10%
2. **3D data** — Design data + material specs → 3D virtual real estate tours (VR without site visit)
3. **CAD data** — Design data → automotive product videos & websites for Toyota, Mercedes pre-launch marketing
4. **Map data** — 800+ staff; AI extracts road/building/signage data; AutoCAD redraw; drone-enhanced 3D since 2018
5. **Books/textbooks (Japanese)** — OCR + image processing + keyword indexing → ePub for mobile; manga + educational content
6. **Medical records & drug catalogs** — OCR scan → Excel index by multiple criteria; prescription app + kiosk drug dispensing integration
7. **Handwritten drafts** — OCR Japanese handwriting → digital; HTML5/epub3/In-design/SVG output
8. **Math textbooks (web)** — OCR + animated math graphics → HTML5/epub3/In-design; Japanese elementary/secondary schools
9. **Intellectual property files** — Digitize and classify copyright certificates; PDF filing system
10. **Urban planning files** — Digitize zoning/construction drawings (large format)
11. **Accounting vouchers** — Scan + OCR invoices/receipts → classify by case/folder; integrate with contract management software; data sync + access control
